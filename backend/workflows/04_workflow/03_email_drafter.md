# Workflow 3: Email Drafter

Reads `recruiters/list.csv`, combines each company's email template with the recruiter's first name, attaches the company-specific resume, and saves the composed email as a **Gmail draft** — without sending. Lets you review every email in Gmail before deciding to send.

**Entry point:** `draft_emails.py`

---

## When to Use

- After running Workflow 3 (email template generation) and Workflow 2 (resume tailoring)
- Before Workflow 6 (email sender) — draft first, review in Gmail, then send

---

## Prerequisites

1. `recruiters/list.csv` — contacts with `Emailed? = No`
2. `output/emails/<Company>_TEMPLATE.txt` — generated by Workflow 3
3. `coop/Justin_Mak_s_Resume_<Company>.pdf` — generated by Workflow 2
4. `.env` configured with Gmail credentials:
   ```
   SMTP_USERNAME=jp.mak44@gmail.com
   SMTP_PASSWORD=<gmail app password>
   FROM_EMAIL=jp.mak44@gmail.com
   FROM_NAME=Justin Mak
   ```
   The same Gmail app password used for sending is also used for IMAP draft access.

---

## How to Run

```bash
# Preview what would be drafted (no connection to Gmail)
python draft_emails.py --dry-run -v

# Create drafts for all Emailed?=No contacts
python draft_emails.py

# Create draft for a single company only
python draft_emails.py --company "Acme Corp"

# Use templates from a specific batch folder
python draft_emails.py --emails-dir output/emails/my_batch

# Debug logging
python draft_emails.py -v
```

---

## Pipeline

```
recruiters/list.csv (Emailed? = blank)
         │
         ▼
  For each unique company:
    Load output/emails/<Company>_TEMPLATE.txt
         │
         ▼
  Replace [RECRUITER_FIRST_NAME] with contact's first name from CSV
         │
         ▼
  Select resume:
    1. coop/Justin_Mak_s_Resume_<Company>.pdf  (company-specific, highest priority)
    2. RESUME_PATH_CODING / _AUTOMATION / _DATA_ENTRY  (by job type)
    3. RESUME_PATH  (fallback)
         │
         ▼
  Gmail IMAP APPEND → [Gmail]/Drafts
         │
         ▼
  Open Gmail → Drafts to review
```

---

## Notes

- **Does NOT mark `Emailed? = Yes`** — drafts are not sends.
- One draft per unique company (first matching contact is used for the recipient address).
- The IMAP connection uses `imap.gmail.com:993` (SSL) with the same app password as SMTP.
- Templates are matched case-insensitively on the normalized company name.
- After reviewing drafts, send via Workflow 6 (`send_emails.py` or `schedule_emails.py`).
