# Workflow 5: Email Sender

Reads `recruiters/list.csv`, matches each contact to their company's email template, substitutes their first name, attaches the correct resume, and sends (or schedules) the email. Skips contacts already marked `Emailed? = Yes`.

Two entry points depending on whether you want immediate or scheduled sending:

| Goal                                                 | Script               |
| ---------------------------------------------------- | -------------------- |
| Send immediately                                     | `send_emails.py`     |
| Schedule for next business day at 9:15 AM local time | `schedule_emails.py` |

---

## Prerequisites

1. `recruiters/list.csv` — contacts with `Emailed? = No` for the ones to send
2. `output/emails/<Company>_TEMPLATE.txt` — generated by Workflow 1 or 2
3. `.env` configured with SMTP credentials and resume paths (see below)

### Required `.env` fields

```
# SMTP
SMTP_SERVER=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=jp.mak44@gmail.com
SMTP_PASSWORD=<app password>
FROM_EMAIL=jp.mak44@gmail.com
FROM_NAME=Justin Mak

# Safety
DRY_RUN=true   # set to false only when ready to send for real

# Resume paths (used automatically based on job type)
RESUME_PATH=coop/Justin_Mak_s_Resume_Automation (2).pdf  # fallback
RESUME_PATH_CODING=coop/Justin_Mak_s_Resume_Coding.pdf
RESUME_PATH_AUTOMATION=coop/Justin_Mak_s_Resume_Automation (2).pdf
RESUME_PATH_DATA_ENTRY=coop/Justin_Mak_s_Resume_Data_Entry.pdf
```

## How to Run

### Immediate send (`send_emails.py`)

```bash
# Preview what would be sent — no SMTP connection (safe default)
python send_emails.py recruiters/list.csv --dry-run -v

# Preview campaign stats only (template availability, counts)
python send_emails.py recruiters/list.csv --preview

# Live send — prompts for confirmation
python send_emails.py recruiters/list.csv --no-dry-run

# Live send — skip confirmation prompt
python send_emails.py recruiters/list.csv --no-dry-run -y
```

### Scheduled send (`schedule_emails.py`)

```bash
# Preview scheduled times per recipient timezone (no emails sent)
python schedule_emails.py recruiters/list.csv --preview

# Schedule + start daemon (required for live sends)
python schedule_emails.py recruiters/list.csv --no-dry-run --start-daemon

# Schedule + start daemon, skip confirmation
python schedule_emails.py recruiters/list.csv --no-dry-run --start-daemon -y

# Custom send time (default is 09:15)
python schedule_emails.py recruiters/list.csv --no-dry-run --start-daemon --send-time 10:00

# Custom stagger between emails in same timezone (default 5s)
python schedule_emails.py recruiters/list.csv --no-dry-run --start-daemon --stagger 10
```

> The scheduler daemon runs in the foreground. Press `Ctrl+C` to stop it. Emails are only sent while the daemon is alive, so keep the terminal open until all scheduled times have passed (or run it in a detached session).

## How It Works

```
Read CSV
  │  skip contacts where Emailed? = Yes
  ▼
For each remaining contact:
  │
  ├─ find output/emails/<Company>_TEMPLATE.txt
  │    case-insensitive match on normalized company name
  │
  ├─ determine job type:
  │    1. CSV "Job Type" column  (explicit)
  │    2. template's JOB TITLE field keyword match  (inferred)
  │       "data entry" / "clerk" → data_entry
  │       "software" / "engineer" / "developer" → coding
  │       anything else → automation
  │
  ├─ select resume:
  │    Select resume based on the company of the resume (match the template's company with the company in the resume name)
  │    (falls back to RESUME_PATH if typed path not configured)
  │
  ├─ replace [RECRUITER_FIRST_NAME] in subject + body
  │
  └─ send via Gmail SMTP (or schedule with APScheduler)
       on success → mark contact Emailed? = Yes in CSV
```

### Resume Selection Logic

The correct resume is picked automatically — you do not need to pass `--resume` manually. Priority:

1. `Job Type` column in CSV (most explicit)
2. Keyword match against the `JOB TITLE` field inside the template file
3. Fallback to `RESUME_PATH` from `.env`

### Scheduling Details (scheduler only)

- Emails are grouped by recipient timezone (mapped from the `Country` column)
- Within each timezone group, emails are staggered by `--stagger` seconds (default 5s)
- Weekends and US federal holidays are skipped automatically
- Target send time is 9:15 AM in the recipient's local timezone by default

---

## CSV Format

```
Email, Company, Level, Country, Emailed?, First name, Job Type
jane@company.com, Acme Corp, Early, California, No, Jane, Automation
```

- `Emailed?` — `Yes` / `No`. The sender sets this to `Yes` after a successful send.
- `Job Type` — optional. If present, overrides the keyword-inferred type. Values: `Coding`, `Automation`, `Data Entry` (case-insensitive).
- `Country` — used by the scheduler to pick the right timezone for send time calculation.

## Safety

- `DRY_RUN=true` in `.env` is the default — nothing is sent without explicitly passing `--no-dry-run`
- Live mode always prints a warning and asks for confirmation unless `--yes` / `-y` is passed
- For `schedule_emails.py`, `--start-daemon` is also required for live mode (prevents accidental scheduling without a running daemon)
- Rate limit: 5 seconds between emails in immediate mode to avoid spam filters
